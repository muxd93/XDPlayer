# 08 - 部署指南

## 8.1 构建产出

### APK 位置

构建完成后，APK 位于：

```
app/build/outputs/apk/
├── debug/
│   └── app-debug.apk              # Debug 通用包
└── release/
    ├── app-armeabi-v7a-release.apk  # arm32 分包
    ├── app-arm64-v8a-release.apk   # arm64 分包
    └── app-universal-release.apk   # 通用包
```

### ABI 选择建议

| 设备架构 | 推荐 APK | 说明 |
|---------|---------|------|
| 现代 TV 盒子（64 位） | `app-arm64-v8a-release.apk` | 体积最小，性能最佳 |
| 老 TV 盒子（32 位） | `app-armeabi-v7a-release.apk` | 兼容 32 位设备 |
| 不确定架构 | `app-universal-release.apk` | 包含所有架构，体积最大 |

## 8.2 Release 构建

### 1. 配置签名

在项目根目录创建 `keystore.properties`（不提交到 Git）：

```properties
storeFile=your_keystore.jks
storePassword=your_store_password
keyAlias=your_key_alias
keyPassword=your_key_password
```

> ⚠️ 当前 `app/build.gradle.kts` 未配置签名读取逻辑，Release 构建默认不签名。
> 如需签名安装，需手动添加签名配置或使用 `apksigner` 工具。

### 2. 构建命令

```bash
# 构建 Release（未签名）
./gradlew assembleRelease

# 指定 ABI 和版本
./gradlew assembleRelease -PtargetAbi=arm64 -PversionCode=98 -PversionName=1.16.0
```

### 3. 手动签名（如构建未签名）

```bash
# 对齐
zipalign -v -p 4 app-universal-release-unsigned.apk app-aligned.apk

# 签名
apksigner sign --ks your_keystore.jks --out app-signed.apk app-aligned.apk

# 验证
apksigner verify app-signed.apk
```

## 8.3 安装部署

### ADB 安装

```bash
# 连接设备
adb connect <TV_IP>:5555

# 安装（-r 覆盖安装，-d 允许降级）
adb install -r app-arm64-v8a-release.apk

# 安装到指定设备
adb -s <device_serial> install -r app.apk
```

### 批量部署脚本

项目提供了辅助编译脚本，可查看其用法：

- Linux/macOS: [build.sh](../build.sh)
- Windows PowerShell: [build.ps1](../build.ps1)

常用示例：

```powershell
# Windows PowerShell
.\build.ps1                                    # 默认 release arm64
.\build.ps1 -Type debug -Abi arm64             # debug 仅 arm64
.\build.ps1 -Type release -Abi arm64 -Split    # release 拆分 APK
.\build.ps1 -Type release -VersionName 2.0.0 -VersionCode 100
```

```bash
# Linux/macOS
./build.sh -t release -a arm64 -s
```

## 8.4 Launcher 模式部署

### 设为默认桌面

1. 安装 APK 后，按 TV 遥控器 Home 键
2. 系统弹出桌面选择器
3. 选择 "MzDKPlayer" 并设为"始终"

### AndroidManifest 配置

[AndroidManifest.xml](../app/src/main/AndroidManifest.xml) 已声明 Launcher 能力：

```xml
<intent-filter>
    <action android:name="android.intent.action.MAIN" />
    <category android:name="android.intent.category.LAUNCHER" />
    <category android:name="android.intent.category.LEANBACK_LAUNCHER" />
    <category android:name="android.intent.category.HOME" />
    <category android:name="android.intent.category.DEFAULT" />
</intent-filter>
```

### 行为说明

| 场景 | 行为 |
|------|------|
| 已设为 Launcher + 老人模式 | 开机进入首页，按 Home 键回到首页 |
| 已设为 Launcher + 标准模式 | 开机进入标准模式主页 |
| 未设为 Launcher | 正常 App 行为 |

## 8.5 Web 配置服务部署

### 自动启动

Web 配置服务在 `MainActivity.onCreate` 中自动启动（当 `webConfigEnabled == true` 时）：

```kotlin
(application as MzDkPlayerApplication).startWebConfigServerIfNeeded()
```

### 访问方式

1. 确保手机/电脑与 TV 在同一局域网
2. 获取 TV 的 IP 地址（TV 设置 → 网络）
3. 浏览器访问：`http://<TV_IP>:18080`

### 端口冲突处理

如端口 18080 被占用，`WebConfigServer.start()` 会抛异常并被 catch，服务不会启动。需检查并释放端口。

## 8.6 权限配置

### 运行时权限

[AndroidManifest.xml](../app/src/main/AndroidManifest.xml) 声明的权限：

| 权限 | 用途 | 类型 |
|------|------|------|
| INTERNET | 网络播放/TMDB API | 普通（自动授予） |
| ACCESS_NETWORK_STATE | 网络状态检测 | 普通 |
| READ_EXTERNAL_STORAGE | 读取本地文件 | 危险（需运行时请求） |
| MANAGE_EXTERNAL_STORAGE | 所有文件访问（Android 11+） | 特殊 |
| WRITE_EXTERNAL_STORAGE | 写入外部存储 | 危险 |
| READ_MEDIA_AUDIO | 读取音频文件（Android 13+） | 危险 |
| RECORD_AUDIO | 麦克风（远程输入语音） | 危险 |

### 存储权限

应用使用 `requestLegacyExternalStorage="true"` 兼容旧版存储模型，同时声明了 `MANAGE_EXTERNAL_STORAGE` 用于 Android 11+ 的全文件访问。

## 8.7 混淆配置

[proguard-rules.pro](../app/proguard-rules.pro) 配置：

- Release 构建启用 `isMinifyEnabled = true` 和 `isShrinkResources = true`
- 需保留的类：
  - Retrofit 接口（`TmdbApiService`）
  - Gson 序列化模型（`*Connection`、`DanmakuResponse` 等）
  - Room 实体
  - JNI 调用的原生方法
