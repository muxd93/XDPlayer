# 09 - 常见问题与已知问题

## 9.1 已知 Bug

### 🔴 P0：AudioPlayerState.observe() 控件无法自动隐藏

**文件**：`app/src/main/java/org/mz/mzdkplayer/ui/audioplayer/components/AudioPlayerState.kt:53`

**现象**：音频播放器控制栏永远不会自动隐藏。

**原因**：`debounce` 后应设为 `false`，但写成了 `true`：

```kotlin
suspend fun observe() {
    channel.consumeAsFlow()
        .debounce { it.toLong() * 1000 }
        .collect { _controlsVisible = true }   // ❌ 应为 false
}
```

**修复**：改为 `_controlsVisible = false`，与 `VideoPlayerState.kt:53` 保持一致。

---

### 🔴 P0：全局 Map 传值导致进程回收崩溃

**文件**：[MzDkPlayerApplication.kt](../app/src/main/java/org/mz/mzdkplayer/MzDkPlayerApplication.kt)

**现象**：音频播放列表通过 `companion object` 的 `stringListMap` 静态 Map 传递，进程被回收后状态丢失。

**影响**：后台被杀后恢复时，播放列表为空。

**建议**：用 `SavedStateHandle` / Navigation 参数 / 单例 Repository + Flow 替代静态 Map。

---

### 🟠 P1：初始化顺序耦合脆弱

**文件**：[RepositoryProvider.kt](../app/src/main/java/org/mz/mzdkplayer/di/RepositoryProvider.kt) + [SettingsRepository.kt](../app/src/main/java/org/mz/mzdkplayer/data/repository/SettingsRepository.kt)

**现象**：
- `SettingsRepository.prefs` 是 `lateinit var`，若 `MainActivity.onCreate` 中 `SettingsRepository.init(this)` 未先调用，任何访问会 `UninitializedPropertyAccessException` 崩溃。
- `RepositoryProvider.database` 为 null 时抛 `IllegalStateException`。

**规避**：确保 `MainActivity.onCreate` 中初始化顺序正确：

```kotlin
RepositoryProvider.init(this)        // 必须先调用
SettingsRepository.init(this)       // 必须先调用
```

---

### 🟠 P1：DataSource 静态资源泄漏风险

**文件**：[SMBDataSource.kt](../app/src/main/java/org/mz/mzdkplayer/tool/SMBDataSource.kt)、[FTPDataSource.kt](../app/src/main/java/org/mz/mzdkplayer/tool/FTPDataSource.kt)、[WebDavDataSource.kt](../app/src/main/java/org/mz/mzdkplayer/tool/WebDavDataSource.kt)

**现象**：三个 DataSource 都在 `companion object` 持有静态连接缓存，播放页退出时需手动调用 3 个 `releaseGlobalResources()` 静态方法。漏调任一即泄漏 socket/文件句柄。

**建议**：把连接生命周期绑到 `IMzPlayer` 或 `DisposableEffect`，由播放器统一释放。

---

### 🟡 P2：onNewIntent 不触发 Compose 重组

**文件**：[MainActivity.kt:113-119](../app/src/main/java/org/mz/mzdkplayer/MainActivity.kt#L113)

**现象**：`externalVideoUri` 是 Activity 字段，`onNewIntent` 更新后不会触发已初始化的 Compose 重组。

**影响**：从外部 App 二次分享视频时可能不响应。

**建议**：将 `externalVideoUri` 提升到 ViewModel 或使用 `rememberUpdatedState`。

## 9.2 架构技术债

5 个协议（SMB/FTP/NFS/WebDAV/HTTP）的 Repository + ViewModel + UI 几乎逐字复制，共约 3000+ 行重复代码。

**影响**：协议逻辑增改一处要同步 5 处，极易遗漏。

**建议**：抽象 `ConnectionRepository<T>` 泛型基类 + `ConnectionListViewModel<T>` + `ConnectionListScreen<T>`。

> ✅ **已修复（Repository + ViewModel 层）**：抽象了 [BaseConnectionRepository.kt](../app/src/main/java/org/mz/mzdkplayer/data/repository/BaseConnectionRepository.kt)（泛型基类，封装 SharedPreferences + Gson 存储和 CRUD）和 [BaseConnectionListViewModel.kt](../app/src/main/java/org/mz/mzdkplayer/ui/screen/vm/BaseConnectionListViewModel.kt)（泛型 ViewModel 基类，封装列表加载/增删改/状态管理）。5 个 Repository 和 5 个 ListViewModel 已重写为继承基类，重复代码从 3000+ 行降至约 200 行。UI 层的 `ConnectionListScreen<T>` 泛型化尚未实现。

> **SMB 已先行改造**：SMB 协议新增了 [SmbConnectionManager.kt](../app/src/main/java/org/mz/mzdkplayer/tool/SmbConnectionManager.kt)（全局连接管理单例，含指数退避自动重连 + 后台健康巡检）和 [SmbDiscoveryScanner.kt](../app/src/main/java/org/mz/mzdkplayer/tool/SmbDiscoveryScanner.kt)（局域网自动扫描），`SMBConViewModel` 的连接管理已委托给 Manager。其他协议仍使用 ViewModel 内手动管理连接的方式，可参考 SMB 的改造模式。

[MzDkPlayerApplication.kt:47](../app/src/main/java/org/mz/mzdkplayer/MzDkPlayerApplication.kt#L47)：

val evictor = LeastRecentlyUsedCacheEvictor(SettingsRepository.exoCacheSizeMb.toLong() * 1024 * 1024)

## 9.3 常见播放问题排查

### 问题：ExoPlayer 无法播放某视频

**排查步骤**：

1. 尝试切换到 VLC 引擎：`设置 → 播放与视频 → 默认播放器内核 → VLC`
2. 查看 `adb logcat | grep "MzExoPlayer"` 错误信息
3. 检查视频格式是否为 ISO/m2ts（这些会自动用 VLC）

### 问题：播放无声（开启音频直通后）

**原因**：音频设备不支持当前音轨格式（如 TrueHD 直通）。

**解决**：关闭音频直通：`设置 → 音频设置 → 音频透传 → 关闭`

> ExoPlayer 引擎会自动判断设备能力，无需手动干预。音频直通仅对 VLC 引擎生效。

### 问题：SMB 播放卡顿

**排查**：

1. 检查网络带宽（播放界面可显示网速）
2. 高码率视频（70-80G 蓝光原盘）+ 弹幕同时开启会卡顿
3. 尝试关闭弹幕

### 问题：TMDB 无法获取影视信息

**原因**：TMDB 在国内可能需要代理。

**解决**：
1. 配置代理或修改 Hosts
2. 确认 `local.properties` 中 `TMDB_API_KEY` 已正确配置
3. 可手动编辑 TMDB 信息（`EditTMDBInfoScreen`）

### 问题：字幕显示异常

**排查**：

1. 外挂字幕需与视频同名（如 `video.mkv` + `video.srt`）
2. ASS/SSA 字幕由播放器渲染，VLC 引擎支持更完整
3. PGS 蓝光字幕需 VLC 引擎
4. 字幕样式可在 `设置 → 字幕` 调整

## 9.4 开发常见问题

### Q: 如何新增一个网络协议？

1. 在 `tool/` 新增 `XxxDataSource.kt`（实现 `DataSource.Factory`）
2. 在 `data/model/` 新增 `XxxConnection.kt`
3. 在 `data/repository/` 新增 `XxxConnectionRepository.kt`（参考 SMBConnectionRepository）
4. 在 `ui/screen/` 新增连接列表与文件列表 Screen
5. 在 `MzDKPlayerAPP.kt` 新增路由
6. 在 `FileMediaInfo.kt` 的 `builderPlayer()` 新增分支

### Q: 如何修改数据库结构？

1. 在 `AppDatabase.kt` 新增 `MIGRATION_X_Y`
2. 更新 `@Database(version = Y)`
3. 在 `addMigrations()` 添加新迁移
4. 新增字段必须提供 `DEFAULT` 值
5. 详见 [04-数据库设计](04-数据库设计.md) 的迁移规范

### Q: 如何切换默认播放引擎？

```kotlin
// 代码方式
SettingsRepository.defaultPlayer = "vlc"  // 或 "exo"

// UI 方式
// 设置 → 播放与视频 → 默认播放器内核
```

### Q: 如何调试 Web 配置服务？

1. 确保 TV 和开发机在同一局域网
2. TV 上启用 Web 配置（默认启用）
3. 浏览器访问 `http://<TV_IP>:18080`
4. 查看 `adb logcat | grep "WebConfigServer"` 日志

### Q: 如何添加新的老人模式栏位类型？

1. 在 `HomeSlotEntity.kt` 的 `slotType` 新增类型
2. 在 `ElderHomeScreen.kt` 的 `slotGridConfig` 和点击逻辑中处理新类型
3. 在 `WebConfigServer.kt` 的 API 中支持新类型
4. 在 Web 配置页面 HTML 中添加选项

## 9.5 性能注意事项

| 场景 | 注意事项 |
|------|---------|
| 蓝光原盘播放 | 70-80G 文件 + 弹幕可能卡顿，建议关闭弹幕 |
| SMB 高码率 | 网络带宽不足时考虑先下载到本地 |
| 大量媒体刮削 | TMDB API 有速率限制，避免短时间内大量请求 |
| ExoPlayer 缓存 | 可在设置中配置（1-10GB），低端设备建议调小 |
| 老人模式栏位 | 默认 5x3=15 个栏位，24 个栏位时卡片较小，建议 8-15 个 |

## 9.6 代码风格约定

项目现有代码风格（接手开发时请遵守）：

- **命名**：Kotlin 驼峰命名，ViewModel 后缀 `ViewModel`，Repository 后缀 `Repository`
- **UI**：Composable 函数首字母大写，Screen 后缀
- **路由**：使用字符串路由名，参数通过 URL 编码传递
- **状态**：ViewModel 使用 `MutableStateFlow + update` 模式
- **错误处理**：API 调用统一用 `Resource<T>` 封装
- **数据库**：Entity 后缀 `Entity`，DAO 后缀 `Dao`
- **注释**：中文注释为主，关键逻辑有注释
